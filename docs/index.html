The Stack

1. Forecasting Agent

Converts SEC filing data into an 8-quarter, driver-based revenue forecast with Confirmed-vs-Hypothesis commentary. Non-ML, fully explainable — every number traces to a named driver and a citation.

↓
2. Data Layer

dbt + DuckDB turn the agent's output into a refreshable, tested (15 automated tests), dashboarded pipeline (Streamlit).

↓
3. FPA Copilot

An LLM (via OpenRouter) routes plain-English requests to deterministic Python tools — the forecast above, plus variance commentary and reconciliation review. The LLM only routes and summarizes; it never does the financial judgment itself.

A Bug, Caught

Mid-build, the forecast engine was compounding a YoY growth rate as if it were a quarterly rate — a unit confusion that would have projected a company's revenue to grow 5.3x in two years instead of the guided ~20%.

Before fix

Q1 2028 forecast: $1,870.6M (a 5.3x jump from the $351.3M base)

After fix

Q1 2028 forecast: $533.6M (a 52% cumulative gain — ~23%/yr, matching guidance)

The fix: convert the annual rate to its equivalent quarterly compounding rate before applying it — (1 + annual_rate)^0.25 - 1 — so 4 quarters of compounding equal the intended 1-year growth, not a 4x'd one. A regression test now guards specifically against this bug recurring.
